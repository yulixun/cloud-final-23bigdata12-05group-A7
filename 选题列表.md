## 《云计算技术》期末实验大作业（分组选题列表）

### 使用说明（先读）

- **分组人数**：建议 4–5 人/组为主，允许 3 人/组（73 人建议约 15–20 组）。
- **选题方式**：
  - **方式 A：从本清单选题**：任选 1 题即可（清单只提供“方向/难度/交付物”参考）。
  - **方式 B：自拟题（允许且鼓励）**：只要与云计算相关即可（IaaS/PaaS/SaaS、虚拟化、容器、K8s、云原生、可观测、DevOps、安全等）。请先提交“题目 + 范围 + 预期交付物”给老师确认。
- **同题说明**：同题可多组，但实现细节需不同（规模/拓扑/镜像初始化/监控维度/自动化程度等）。
- **最低交付物（所有题目通用）**：
  - **可复现实验环境说明**（硬件/系统/网络/版本/镜像来源）
  - **关键命令与配置**（可截图，但必须配文字解释；建议附命令清单）
  - **功能验证**（用例/预期/实际结果 + 截图/日志/输出）
  - **排错复盘**（≥3：现象-定位-根因-解决-预防）
  - **答辩材料**：8–12 页 PPT（建议结构见 README）

---

### 推荐题目（10 题，覆盖课程主线）

#### 题目 1（OpenStack）Neutron 虚拟网络：网络/子网/端口创建与联通验证（推荐）

- **最低要求**：CLI 创建 network/subnet/port；至少 2 个网络+2 个子网；至少 2 台云主机完成联通验证；完成 1 次“故障注入+排错”。
- **加分点**：给出网络拓扑图；对比 Dashboard 与 CLI；补充安全组/路由解释。

#### 题目 2（OpenStack）云主机与快照：生命周期管理 + 快照回滚演示（推荐）

- **最低要求**：CLI 创建/启停/删除/锁定/暂停等操作；创建快照并用快照重建/恢复云主机；总结“快照 vs 镜像”的使用场景。
- **加分点**：写批量脚本（含错误处理）；给出操作前后差异与风险说明。

#### 题目 3（OpenStack）镜像部署与初始化：Key Pair + cloud-init + 安全组（进阶）

- **最低要求**：创建/管理 key pair 并 SSH 登录；编写 cloud-init（至少完成：用户/公钥/安装服务/写配置其一）；设计并验证安全组最小权限策略。
- **加分点**：实现可重复初始化（多台云主机一致）；补充安全风险与合规说明。

#### 题目 4（虚拟化 vs 容器）同一应用的性能与隔离边界对比实验（进阶）

- **最低要求**：同一应用分别用 VM 与 Docker 运行；对比启动时间、CPU/内存/磁盘占用（≥3 指标）；解释隔离边界差异。
- **加分点**：补充压测或并发对比；结合“快速弹性伸缩”做结论。

#### 题目 5（课程综合）资源池化与可计量服务：设计“按量计费”指标与采集方案（挑战）

- **最低要求**：计算/存储/网络各设计 1 个可计量指标；说明采集方式与数据格式；给出 1 天费用计算示例。
- **加分点**：加入 SLA（可用性/延迟）与告警策略；讨论计量误差与边界。

#### 题目 6（Docker）镜像构建：Dockerfile + 运行参数 + 版本管理（入门）

- **最低要求**：编写 Dockerfile 构建镜像；用 `docker run` 演示端口映射/卷挂载/环境变量；给出 tag 命名规则。
- **加分点**：镜像瘦身（多阶段构建等）并量化体积变化。

#### 题目 7（Docker Compose）三层应用编排：前端 + 后端 + 数据库（入门/进阶）

- **最低要求**：`docker-compose.yml` ≥3 服务；数据库 volume 持久化并验证；说明健康检查或启动顺序控制。
- **加分点**：加入日志采集或简单监控；给出一键启动/停止脚本。

#### 题目 8（Kubernetes）Deployment + Service：滚动升级与回滚演示（进阶）

- **最低要求**：部署应用（Deployment）并暴露服务（Service）；演示滚动升级与回滚；至少配置 1 个探针并解释与可用性的关系。
- **加分点**：补充 ConfigMap/Secret；解释副本数与容量规划。

#### 题目 9（可观测）监控 + 日志 + 告警的最小闭环（挑战）

- **最低要求**：给出指标与采集展示方式；至少 1 条告警规则（阈值/触发/建议）；说明日志采集与查询示例。
- **加分点**：补充故障演练（触发告警并定位）；说明扩缩容策略（HPA/手动均可）。

#### 题目 10（云原生交付）从代码到上线：CI 构建镜像 + 推送 + 部署（挑战）

- **最低要求**：流水线完成构建/（可简化测试）/打包镜像/推送；部署端演示升级与回滚；给出版本追溯链路（运行实例 -> 镜像 tag -> commit）。
- **加分点**：加入安全扫描/依赖检查；补充发布策略（蓝绿/金丝雀思路即可）。
